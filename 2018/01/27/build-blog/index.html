Github + Hexo 搭建属于自己的博客

2018-01-27 blog搭建 0 评论

2018年，决定重新把blog捡回来，利用Github + Hexo 搭建了自己的博客，希望从此能够在这里留下自己更多的足迹。

这里粗略的记录搭建的过程

  • 申请GitHub账号并新建仓库
  • 下载并安装 Git 和 node.js
  • hexo 安装配置
  • 部署到GitHub

1、申请GitHub账号并新建仓库

（1）点击 github，申请属于自己的 Github 账号
（2）新建仓库，如下图，注意Repository name 应和 Owner 处的名字相同

2、下载并安装 Git 和 node.js

（1）git 下载地址

（2）node.js 下载地址

安装完成之后，在命令行窗口（cmd）输入 “node -v” 和 “git –version” 检测是否安装成功，出现对应的版本号则说明安装成功

3、hexo 安装配置

(1) git 和 node.js 安装成功之后，在任意位置点击右键，选择git bash

输入以下命令安装hexo

npm install -g hexo-cli

(2) 输入命令hexo version验证是否安装成功

(3) 安装 Hexo 完成后，新建一个文件夹，进入文件夹，右键运行 git bash,执行下列命令进行初始化，Hexo 将会在该文件夹中新建所需要的文件,关于各文件及文件夹的意义，可参考官方文档：

hexo init 
npm install

(4) 初始化完后，先安装hexo-deployer-git插件，因为后续我们将把代码上传至GitHub：

npm install hexo-deployer-git --save

(5) 修改主目录下的_config.yml，进行基础配置

该文件前面的title、 author 等根据自己的爱好修改，主要配置部署的是最后一项deploy，如下：

deploy:
  type: git
  repository: https://github.com/HJiaoBME/HJiaoBME.github.io.git
  branch: master

其中， repository是上面在GitHub上所新建的仓库地址

4、部署到GitHub

(1) 新建博客

在终端输入：hexo new <文章标题>，即可在本地的hexo初始化生成的source文件夹下的_post 文件夹下看到新建的markdown文件，可以用文本编辑器打开，按照相应的格式编写博客内容。更多请参见写作

(2) 输入一下命令，进行本地发布

hexo s

会出现一个地址，一般是http://localhost:4000/，将其复制到网页即可预览效果。

tips: Hexo 默认端口是4000，若本地打不开，则说明该端口被占用，启动hexo s 的时候，用以下命令，换一个端口
hexo s -p 5000

(3) 在终端执行以下命令，发布到GitHub

hexo g
hexo d

这时博客已经部署到网上了，在浏览器地址输入栏输入blog网址即可，如我的博客是：hjiaobme.github.io
